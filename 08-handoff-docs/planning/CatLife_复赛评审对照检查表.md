# CatLife 复赛评审对照检查表

> 逐条对齐复赛评审标准，确保不遗漏任何评估维度
> 基于：应用赛道复赛交流会-0528.pdf

---

## 一、创新性（40%）— 核心战场

### 1.1 选题定位
- [ ] **从用户真实需求出发**：是否有用户访谈记录？（≥5份）
- [ ] **体现对体验/情感/社会价值的关注**：专注辅助 → 心理成长？陪伴感？
- [ ] **理念清晰完整**：SDT+正念贯穿全流程设计，非事后贴标签

### 1.2 产品功能
- [ ] **原创性**：CatLife与番茄钟/锁机/宠物App的本质区别是什么？
- [ ] **突破传统思维**：AI行为识别不是简单的"一段时间不操作"
- [ ] **功能设计清晰明确**：四状态机+双入口+轻锁定，每个功能都有明确的使用场景

### 1.3 产品界面
- [ ] **完整美观**：所有页面UI完整（开屏/领养/主场景/专注/记录/设置）
- [ ] **易懂易操作**：新手5秒内理解产品用途，不需教程

### 1.4 产品交互
- [ ] **流畅自然**：状态切换无卡顿，动画连贯
- [ ] **符合用户习惯**：手势操作（上滑退出）符合直觉
- [ ] **解决痛点**：上滑退出比锁机App的强制限制更温和
- [ ] **新颖操作体验**：猫咪行为随用户状态变化是亮点

## 二、应用价值（30%）

### 2.1 可行性
- [ ] **技术可实现性较强**：Unity+Android已验证技术栈
- [ ] **具备转化路径**：教育场景（高校/自习室）推广路径清晰

### 2.2 前景评估
- [ ] **目标用户足够广泛**：大学生备考群体，超5000万人
- [ ] **痛点足够明确**：数字分心→学习效率下降→焦虑循环
- [ ] **场景真实高频**：图书馆/宿舍/自习室，日均触发2-3次
- [ ] **独特优势**：AI识别+猫咪陪伴+轻锁定的三重差异
- [ ] **符合行业趋势**：AI+心理健康/教育科技赛道

## 四、大模型应用能力（10%）

### 4.1 理解力
- [ ] **阐释大模型应用场景**：为什么专注判定需要用LLM而不是规则？
- [ ] **场景选择合理**：不是"为了用AI而用AI"

### 4.2 调用能力
- [ ] **通过API调用实现产品功能**：至少1个场景可实际调用
- [ ] **调用代码清晰可复现**：评委能看懂调用逻辑
- [ ] **成本可控**：每30s调1次，token消耗合理
