## 1. 视频定位

官方规格：尽量不超过 3 分钟，最长不超过 5 分钟；推荐 MP4；横版 1920x1080 或竖版 1080x1920。
建议比例：优先竖屏 9:16；如果 PPT 里嵌入，也保留横版裁切图。
核心信息：CatLife 不是普通番茄钟，而是“行为识别 + 猫咪陪伴 + 轻锁定 + 专注复盘”的闭环。

## 3. 3 分钟旁白稿

各位评委老师好，我们的作品是 CatLife，一款基于 AI 行为识别和虚拟猫陪伴的专注辅助软件。

我们解决的不是“用户不会计时”，而是更前面的困难：当用户正在高频刷动、频繁切换、处理碎片信息时，很难自然进入稳定专注。传统番茄钟通常从开始计时之后才工作，CatLife 关注的是从分心到专注的过渡过程。

在主场景中，用户会看到一个低多边形猫咪小镇和一只会响应状态变化的猫。普通状态下，猫保持活跃，用户可以互动；当系统发现操作频率下降、停顿时间增加时，猫会进入过渡动作，靠近、观察或轻轻嗅闻，把用户注意力从高刺激操作中接住。

当用户进入专注状态后，界面会降低干扰，猫也从活跃互动切换成安静陪伴。这里不是强制锁机，用户仍然可以退出，但退出需要一个明确动作。我们希望保留自主性，而不是用压力管理用户。

完成一段专注后，猫会给出奖励反馈，并生成一段温和复盘。大模型部分只处理去标识化的聚合特征，例如专注时长、中断次数和状态序列，不上传原始输入内容。没有网络或没有 API Key 时，也会自动降级为本地模板。

CatLife 的创新点在于把行为识别、猫咪状态、轻锁定交互和心理学中的自主性、胜任感、关系感结合起来。它不是更强硬的控制工具，而是用陪伴式设计帮助用户自然进入专注。

## 5. 录制规范

- 每个镜头单独录，避免一镜到底失败后返工。
- 真机录屏优先使用 `adb screenrecord`，注意它不录音，旁白后期配。
- 每段镜头前后多留 1 秒，方便剪辑。
- 不在视频中展示 API Key、个人账号、系统通知、桌面隐私内容。
- 视频中出现的功能必须在 APK 或 Unity Game View 中真实可复现。

## 7. 验收标准

- 时长满足复赛要求。
- 第一屏 5 秒内看得出产品名和猫咪小镇。
- 至少出现一次完整状态链：普通 -> 过渡 -> 专注 -> 奖励。
- 至少出现一次 APK 或真机证据。
- 必须展示作品最终形态、作品名称、交互 UI 与功能演示。
- 大模型能力有输入输出解释，不夸大为“读取全手机行为”。
- 画面不出现未授权隐私、密钥、聊天窗口。
