# CatLife 用户场景、操作响应模拟与用例覆盖验证

日期：2026-06-29
目标：分析用户可能使用方式和真实需求，验证现有用例库能否覆盖实际场景。

## 1. 用户需求分析

| 用户需求 | 真实含义 | CatLife 响应 |
|---|---|---|
| 我想开始学习但停不下来 | 最大难点在开始前的过渡 | 主小镇 + 猫咪靠近 + 轻提示 |
| 我不喜欢强锁机 | 用户需要自主感 | 保留上滑退出，不强制封锁 |
| 我希望有人陪但不要打扰 | 陪伴要低刺激 | 专注态猫趴下、UI 隐藏 |
| 我想知道自己有没有进步 | 需要反馈但不能压力太大 | 记录页展示时长和稳定度 |
| 我担心隐私 | 不想被监控屏幕/输入 | 设置页明确本地统计和聚合上传 |
| 我断网也想用 | 专注不应依赖云端 | 本地状态机和模板文案 |
| 评审想看技术真实性 | 需要能证明 AI/LLM 边界 | Debug 层 + 代码包 + 隐私网关 |

## 5. 覆盖缺口

| 缺口 | 风险 | 建议 |
|---|---|---|
| 个性化阈值未细化 | 不同用户操作习惯不同 | P2 增加 `UC-21 调整灵敏度` |
| 首次隐私授权流程未细化 | 评审可能关注数据边界 | P1 增加首次引导中的隐私短说明 |
| 记录页过于轻量 | 应用价值表达可能不足 | P1 增加 7 天趋势和会话洞察 |
| LLM 真实调用失败演示 | 大模型应用分可能受影响 | 必须准备成功日志或 mock/真实双路径 |

## 7. 最终判断

当前用例库已经覆盖 CatLife 的核心需求：

```text
打开 App -> 进入小镇 -> 开始专注 -> 行为识别 -> 猫状态变化 -> 轻锁定 -> 奖励 -> 记录 -> 隐私/LLM 解释
```

需要补强的不是页面数量，而是：

1. 首次隐私说明；
2. 记录页趋势表达；
3. LLM 成功和失败两套证据；
4. 比赛演示证据导出。

这些补强项都不需要新增活动、商城、抽奖或重置页面，仍然贴合 CatLife 的专注陪伴主题。
