## 2. 评分口径

| 维度 | 权重 | 官方关注点 | CatLife 对齐策略 |
|---|---:|---|---|
| 作品创新性 | 40% | 选题定位、产品功能、产品界面、产品交互 | 强化“专注过渡阶段 + 猫咪状态反馈 + 轻锁定”差异化 |
| 应用价值 | 30% | 可行性、前景评估 | 用 APK 真机运行、大学生学习场景和推广路径证明 |
| 作品完成度 | 20% | 资料齐全度、资料质量 | 5 项材料齐全，且 PPT/视频/海报必须用真实效果 |
| 大模型应用能力 | 10% | 阐释应用场景；通过 API 调用实现产品功能 | 至少一个可演示模型调用或端侧/云端调用 Demo，代码包清晰标注 |

## 3. 经验分享转化为 CatLife 约束

| 方法论 | 对 CatLife 的约束 |
|---|---|
| Design Thinking | PPT 不能只堆功能，要按 Discover/Define/Develop/Deliver 讲清楚为什么选大学生专注场景 |
| 场景聚焦 | 不扩展成泛心理陪伴、社交、养老、儿童陪伴；只讲“大学生从分心到专注” |
| 用户验证 | 至少补 5 份访谈或问卷摘要，否则“用户真实需求”证据弱 |
| 工程化 | Demo 不完整是红线；APK 可运行优先级高于继续堆美术效果 |
| 差异化 | 必须讲清楚不是番茄钟、不是强锁机、不是普通宠物 App |
