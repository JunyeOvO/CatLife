|------|------|----------|--------------------------|

| 应用价值 | 30% | 场景洞察 | **可行性**+前景评估 |

### 创新性详细评估标准（40分）
- 选题定位：从用户真实需求出发，理念贯穿设计全过程
- 产品功能：**原创性**，突破传统思维，功能设计清晰
- 产品界面：完整美观，易懂易操作
- 产品交互：流畅自然，解决痛点，提供新颖操作体验

### 应用价值评估标准（30分）
- 可行性：技术可实现性较强，具备转化路径
- 前景评估：目标用户广泛、痛点明确、场景真实高频；市场独特优势

### 完成度评估标准（20分——初赛不存在）
- 资料齐全度：全部5项按要求提交
- 资料质量：内容完整度、丰满度、呈现效果

### 大模型应用评估（10分）
- 有阐释大模型应用的场景说明
- **通过API调用实现产品功能**

## 四、创新创意经验分享 · 关键方法论

> 分享人：顾卫华（vivo，连续三届评委）

### 4.1 核心框架：Design Thinking

```
Discover(发现问题) → Define(定义问题) → Develop(发想方案) → Deliver(交付方案)
```

对应工具：
- Discover: PEST分析、用户画像Persona、用户旅程图
- Define: JBTD(Jobs to be Done)、HMW(How Might We)、Five Whys、POV
- Develop: Crazy 8's、原型设计、MVP
- Deliver: Demo、A/B Test、KPI验证

### 4.2 创新验证方法论（评委最看重的）

必须回答的四个核心问题：
1. **定位**：为谁解决什么问题？→ 观察、假设
2. **必要性**：解决的必要性有多大？→ **定性+定量分析**
3. **方案**：如何解决问题？→ **差异化解决方案**
4. **呈现**：解决方案是否简洁清晰有惊喜感？→ 聚焦核心流程

### 4.3 评委淘汰案例（红线）

| 问题类型 | 具体表现 | CatLife自查 |
|----------|----------|-------------|
| 场景贪多 | 情绪疏导+孩子陪伴+社交恐惧+独居老人+个性化 → 通用AI助手更合适 | ✅ 已聚焦大学生专注场景 |
| 功能复杂 | 功能定义不直观、方案设计复杂 → 用户学习成本高 | ⚠️ 需确保交互简洁 |
| 无数据验证 | 有人群假设但无访谈验证 / 有定性说明无定量分析 | ⚠️ 需补用户验证数据 |
| 同质化 | 没讲出侧重点，没讲清楚解决哪个具体问题 | ⚠️ 需强化差异化论述 |
| 工程化不足 | Demo不完整，达不到演示标准 | ⚠️ 核心风险！ |

### 4.4 评委建议

1. **避免同质化**：同类想法要讲出自己的侧重点和独特方案
2. **工程化**：Demo必须完备，达到可演示标准
3. **跳出舒适区**：不要用过往项目参赛
4. **全流程用AI**：从想法验证→方案设计→落地，全部结合AI工具
5. **AI工具推荐**：APP生成、高保真设计生成、pm-skills产品分析、文案/素材生成

### 4.5 推荐书目

- 《The Design Thinking Playbook》— 斯坦福创新方法论
- 《设计思维手册》— 产品/服务/商业数字化转型
- 《以用户为中心的产品设计》

## 五、复赛提供的工具/能力（视频转录完整版）

> 来源：复赛交流会-录屏.mp4（94分钟完整转录）
> 分享人：黄中业（云真机）+ 初心严（端侧部署）

### 5.1 云真机（Cloud Real Device）

**是什么**：vivo提供的远程真实手机调试平台，部署在机房服务器上。

| 特性 | 详情 |
|------|------|
| 机型 | vivo X300 Pro（真实手机，非模拟器） |
| 系统 | Android 16，特殊版本内置AIGC调试能力 |
| 访问方式 | Web网页端 → 点击"开始调试" → 随机分配一台手机 |
| 时长限制 | 每次8小时，超时自动释放（释放后可重新申请） |
| 数量 | 有限，满额需排队等待 |
| 权限 | 仅队长手机号可登录 |
| 数据 | 释放后数据清空 |

**支持功能**：
- APK上传安装（网页拖拽即可）
- ADB远程连接（复制命令→终端执行→Android Studio直接调试）
- 实时性能监控
- 截图、录屏（可保存到本地电脑）
- WiFi、横竖屏切换
- 真实相机（但机房固定视角，无运动变化）
- 不支持：麦克风输入、Root权限

**ADB连接示例**：`adb connect <云真机提供的IP:端口>` → `adb devices` 确认 → Android Studio识别

**重要**：所有进入复赛的队伍将获得一台**真实vivo手机**用于调试（除云真机外）。

### 5.2 端侧大模型部署（On-Device AI）

**蓝心3B 大模型**（BlueLM 3B）：

| 特性 | 详情 |
|------|------|
| 模型 | 蓝心3B（前几届7B，本届降为3B以优化端侧效率） |
| 类型 | 纯文本模型（不支持多模态） |
| 部署位置 | 完全在手机上运行，不需要网络 |
| SDK | Java/C++ API，已内置在云真机系统中 |
| 调用方式 | 极简：初始化→喂Prompt→流式返回Token→释放 |
| Demo | 官方提供Demo APK + 源码，可直接安装体验 |
| 附加能力 | 文本审核SDK（敏感内容过滤） |

**端侧优势 vs 劣势**：
| 优势 | 劣势 |
|------|------|
| 🔒 隐私安全（数据不出手机） | 📉 模型能力不如云端大模型 |
| ⚡ 低延迟（无网络往返） | 📏 参数小（3B vs 云端数百B） |
| 📡 离线可用 | 🔧 精度可能因量化压缩降低 |
| 💰 零推理成本 | |

**适用场景建议**：
- ✅ 端侧适合：敏感信息处理、实时响应、离线场景
- ✅ 云端适合：复杂推理、大参数需求
- 💡 **最佳实践**：云端+端侧混合 —— 隐私敏感的用端侧，复杂功能用云端

**复赛阶段**：提供通用3B模型 + SDK + Demo源码
**决赛阶段**：提供微调平台，可上传数据对模型做定制改造

**关键澄清**：
- ❌ 端侧模型不是强制要求 —— 可以用任何第三方API（DeepSeek/豆包/OpenAI等）
- ❌ 使用云真机不等于必须用端侧模型 —— 可以只在云真机上调试APK
- ✅ 端侧模型是"加分项"而非"必选项"
- ✅ Demo源码可直接用AI工具理解改造（把文档丢给AI即可）

### 5.3 对CatLife的启示

| 端侧能力 | CatLife可应用的场景 | 价值 |
|----------|---------------------|------|
| 蓝心3B文本模型 | 猫咪对话（简短回复<50字） | 隐私安全，对话内容无需上传云端 |
| 蓝心3B文本模型 | 专注状态文字判定 | 低延迟实时反馈 |
| 文本审核SDK | 用户输入过滤 | 安全合规 |
| 云端大模型 | 复杂专注分析、总结生成 | 高精度推理 |

> **推荐策略**：混合架构 —— 蓝心3B做实时猫咪对话（低延迟+隐私），DeepSeek做复杂专注分析（高精度）。在提交材料中展示"端侧+云端双模型协同"可突出大模型应用能力（10%）和创新性（40%）。
