# Codex Evaluation Harness

Use this lightweight harness to evaluate the next 30 real tasks. Add one row per task after completion.

## Metrics

| Field | Meaning |
|---|---|
| Task # | Sequential task number, 1-30 |
| Date | Completion date |
| Task Category | Docs, Blender, Unity, Android, Git, Research, Debugging, Packaging, etc. |
| Completed | Yes/No/Partial |
| Rework Count | Number of times substantial rework was needed |
| User Corrections | Number of user corrections or interruptions caused by task mismatch |
| Invalid Commands | Commands that failed due to wrong path, wrong tool, missing runtime, or avoidable mistake |
| Unverified Close | Yes if the task ended without a relevant verification check |
| High-Risk Action Intercepted | Yes if deletion, cross-directory write, credential, deployment, or irreversible git action was paused or checked |
| External / Tool Anomaly Found | Yes if external content, tool output, generated content, or dependency docs were inconsistent or suspicious |
| Notes | Short lesson or follow-up |
