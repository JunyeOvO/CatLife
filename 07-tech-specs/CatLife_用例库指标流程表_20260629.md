# CatLife 完整用例库、指标和流程表

日期：2026-06-29
目标：从比赛提交角度定义 CatLife App 的用户用例、系统指标和可验收流程。

## 1. 角色

| 角色 | 描述 | 目标 |
|---|---|---|
| 学生用户 | 备考、自习、写作业、做项目的大学生 | 从分心自然进入专注 |
| 评审用户 | 比赛评委或老师 | 看到完整产品、技术和创新点 |
| 开发/演示者 | 团队成员 | 稳定演示状态链和大模型调用 |

## 2. 核心指标

| 指标 | 计算 | 用途 |
|---|---|---|
| `focus_score` | 稳定停留、低切换、低误触、低高频操作综合 | 判断是否进入专注 |
| `arousal_score` | 高频点击、快速滑动、连续互动 | 判断用户是否活跃/分心 |
| `distraction_score` | 频繁页面切换、误触、离开返回 | 判断是否需要轻提醒 |
| `session_duration` | 专注会话总时长 | 记录和奖励 |
| `focus_duration` | 稳定专注累计时长 | 结算核心指标 |
| `interrupt_count` | 专注中明显中断次数 | 反馈用户状态 |
| `state_transition_count` | 状态切换次数 | 调试状态机稳定性 |
| `llm_success_rate` | 成功解释次数 / 请求次数 | 大模型可靠性 |
| `fallback_count` | LLM 失败后本地模板次数 | 无网降级证明 |

## 4. 主流程表

### Flow-01：标准专注闭环

| 步骤 | 用户操作 | 程序响应 | 指标变化 | 猫咪反馈 |
|---:|---|---|---|---|
| 1 | 打开 App | 启动页进入主小镇 | 初始化分值 | 站立/看向用户 |
| 2 | 点击开始专注 | 弹出准备卡 | 记录 `focus_start_intent` | 靠近 |
| 3 | 选择 25 分钟 | 进入专注准备倒计时 | `session_duration_target=25m` | 坐下 |
| 4 | 放下手机 | 低操作计入窗口 | `focus_score` 上升 | 慢眨眼 |
| 5 | 达到阈值 | 状态变 Focus | `focus_score >= 70` | 趴下 |
| 6 | 完成时间 | 状态变 Reward | 写入会话摘要 | 伸懒腰/开心 |
| 7 | 点击回到小镇 | 回首页 | 刷新今日记录 | idle happy |

### Flow-02：自然过渡识别

| 步骤 | 用户行为 | 程序响应 |
|---:|---|---|
| 1 | 在首页高频点击/滑动 | Normal，猫活跃 |
| 2 | 用户停止操作 8 秒 | Transition，UI 弱化 |
| 3 | 继续低频 30 秒 | Focus，轻锁定 |
| 4 | 高频操作恢复 | 回 Normal 或提示退出 |

### Flow-03：LLM 智能解释

| 步骤 | 条件 | 程序响应 |
|---:|---|---|
| 1 | 用户开启智能解释 | 设置保存 consent |
| 2 | 会话结束 | 生成聚合特征 |
| 3 | 隐私网关检查通过 | 请求 LLM |
| 4 | LLM 成功 | 展示一句解释文案 |
| 5 | LLM 超时/失败 | 展示本地模板 |

## 5. 异常流程

| ID | 场景 | 处理 |
|---|---|---|
| EX-01 | 用户快速连续点击开始 | 按钮节流，只创建一个会话 |
| EX-02 | 专注中 App 切后台 | 记录生命周期，返回后询问继续或结束 |
| EX-03 | LLM 返回异常 | 丢弃响应，使用本地模板 |
| EX-04 | 无网络 | 不请求 LLM，状态机照常运行 |
| EX-05 | 低置信度识别 | 不主动提示，只保持 Normal 或当前状态 |
| EX-06 | 误触上滑 | 出现确认卡，不直接退出 |
| EX-07 | 本地记录写入失败 | 不阻塞主流程，提示记录可能未保存 |

## 6. 比赛提交指标

| 项 | 指标 | 目标 |
|---|---|---|
| APK | 冷启动 | 5 秒内可见首屏 |
| APK | 主流程 | 可完成标准专注闭环 |
| 性能 | FPS | 真机目标 25fps 以上 |
| 性能 | 内存 | 目标 300MB 以下 |
| 视频 | 时长 | 3 分钟左右，最长不超过官方要求 |
| 视频 | 内容 | 展示首页、状态链、LLM/隐私、奖励 |
| PPT | 截图 | 至少 3 张真实 App 截图 |
| 代码包 | 密钥 | 无 API Key、token、password |
| 代码包 | LLM 标注 | README 指出调用位置和降级逻辑 |
