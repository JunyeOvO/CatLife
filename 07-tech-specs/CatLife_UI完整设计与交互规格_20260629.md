## 1. 产品 UI 定位

CatLife 的 UI 不是效率工具仪表盘，也不是传统宠物养成游戏。它的界面目标是：

```text
用低刺激界面承接用户从分心到专注的过渡，让猫咪状态成为主要反馈。
```

设计原则：

| 原则 | 说明 |
|---|---|
| 低刺激 | 专注前减少红点、弹窗、强提醒，不做连续任务诱导。 |
| 猫咪优先 | 主画面先看见猫和小镇，UI 只作为辅助层。 |
| 状态可理解 | 用户一眼知道猫为什么活跃、靠近、趴下或奖励。 |
| 自主退出 | 专注态保留上滑退出，不做强制锁机。 |
| 隐私可见 | 明确说明只使用本 App 内交互节奏和聚合特征。 |
| 比赛可展示 | 每个核心页面都能进入 PPT/视频截图。 |

## 2. 视觉风格

| 项 | 设计 |
|---|---|
| 风格关键词 | 低多边形、暖色小镇、柔和陪伴、轻学习、童话但不幼稚 |
| 主色 | 暖橙 `#F59A45`，来自橘猫和屋顶 |
| 辅色 | 草地绿 `#6CAB5F`，天空蓝 `#9CC9E8`，奶白 `#FFF3E2` |
| 功能色 | 专注蓝绿 `#55AFA6`，奖励金 `#F5C85B`，提醒棕 `#8A6042` |
| 字体 | 中文优先使用系统无衬线；按钮和数字保持高可读性 |
| 图标 | 线性圆角图标，猫爪、时钟、叶子、月亮、设置 |
| 动效 | 轻微呼吸、滑入、淡入；不做强粒子、不做高速闪烁 |

## 3. 信息架构

```text
启动页
  -> 主小镇页
      -> 专注准备卡
      -> 专注进行层
      -> 奖励结算层
      -> 记录页
      -> 猫咪状态页
      -> 隐私与智能解释设置
```

不设计的页面：

| 不做 | 原因 |
|---|---|
| 活动中心 | 与复赛产品闭环无关，会稀释专注主题。 |
| 抽奖/签到 | 会把产品变成游戏化运营，而不是专注辅助。 |
| 重置活动 | 没有真实业务价值。 |
| 商城 | MVP 不需要。 |
| 社交排行 | 可能制造比较压力，违背温和陪伴定位。 |

## 4. 页面 01：启动页

### 目标

5 秒内让评审和用户知道这是 CatLife，并看到猫咪小镇气质。

### 布局

| 区域 | 内容 |
|---|---|
| 背景 | 小镇全景或模糊低多边形天空/草地，不用纯渐变 |
| 中央 | CatLife 标识 + 橘猫剪影/小爪印 |
| 底部 | `让专注自然发生`，进入按钮 |

### 交互

| 用户操作 | 程序响应 |
|---|---|
| 点击进入 | 淡入主小镇页，猫从 idle 状态开始 |
| 停留 2 秒 | 自动显示进入按钮 |

## 5. 页面 02：主小镇页

### 目标

这是 App 的默认首页，也是视频和 PPT 最重要截图。小镇和猫必须是第一视觉中心。

### 布局

```text
顶部：今日状态小条
中部：3D 小镇 + 猫咪
底部：开始专注 / 今日记录 / 猫咪状态
右上：设置
```

### 内容

| 组件 | 文案/数据 | 说明 |
|---|---|---|
| 状态小条 | `今天已安静陪伴 18 分钟` | 不用强 KPI，语气柔和。 |
| 猫咪气泡 | `先不用急，我在这里。` | 本地模板或 LLM 文案。 |
| 主按钮 | `开始专注` | 主要行动。 |
| 次按钮 | `记录`、`猫咪` | 使用图标 + 短文字。 |

### 状态变化

| 状态 | UI | 猫 |
|---|---|---|
| 普通 | 按钮完整、状态条可见 | 站立、摆尾、看向用户 |
| 过渡 | 底部按钮弱化，提示变轻 | 靠近、坐下、慢眨眼 |
| 专注 | 隐藏非必要按钮，只保留上滑退出 | 趴下、呼吸 |
| 奖励 | 显示完成卡片 | 伸懒腰、开心 |

## 6. 页面 03：专注准备卡

### 目标

让用户自主开始，不制造压力。

### 布局

弹出底部半屏卡，不遮住猫头和主体。

| 字段 | 选项 |
|---|---|
| 本次目标 | 15 分钟、25 分钟、45 分钟、自定义 |
| 模式 | 安静陪伴、轻提醒 |
| 智能解释 | 开关，默认关闭或按用户设置 |

### 文案

```text
我会把动作放慢，陪你进入这段时间。
```

按钮：

- `开始`
- `稍后`

## 7. 页面 04：专注进行层

### 目标

减少干扰，让 UI 退到背景，猫咪成为低频陪伴反馈。

### 布局

| 区域 | 内容 |
|---|---|
| 顶部 | 小号倒计时，如 `18:42` |
| 中央 | 猫咪趴下或安静呼吸 |
| 底部 | `上滑退出` 提示 |

不显示：

- 活动入口；
- 多余菜单；
- 复杂数据曲线；
- 强提醒弹窗。

### 状态反馈

| 识别结果 | UI 响应 | 猫响应 |
|---|---|---|
| 稳定专注 | 计时保持，画面更安静 | 呼吸循环 |
| 轻微分心 | 出现一句弱提示 | 抬头看一下 |
| 频繁操作 | 询问是否退出或回到任务 | 坐起 |
| 用户上滑 | 显示确认卡 | 猫看向用户 |

## 8. 页面 05：奖励结算层

### 目标

把完成感转化为温和反馈，不做过度游戏化。

### 内容

| 组件 | 示例 |
|---|---|
| 标题 | `完成了一段安静时间` |
| 数据 | `专注 25 分钟`、`中断 1 次`、`稳定度 82` |
| 猫反馈 | `猫咪在你旁边睡着了` |
| 奖励 | 小爪印、猫咪心情 +1 |
| 操作 | `回到小镇`、`再来一段` |

## 9. 页面 06：记录页

### 目标

展示趋势，不制造打卡压力。

### 模块

| 模块 | 内容 |
|---|---|
| 今日 | 专注总时长、完成段数、最长稳定段 |
| 最近 7 天 | 柔和柱状图或小镇足迹 |
| 会话列表 | 时间、时长、猫咪反馈 |
| 洞察 | 本地模板或 LLM 解释 |

文案示例：

```text
你在晚上更容易进入稳定状态，可以把难一点的任务放在这个时间。
```

## 10. 页面 07：猫咪状态页

### 目标

解释猫的行为和用户状态的关系，让产品差异化可见。

### 内容

| 组件 | 内容 |
|---|---|
| 猫咪当前心情 | 安静、好奇、陪伴、开心 |
| 动作说明 | `趴下：你已经进入稳定低操作状态` |
| 成长值 | 只展示陪伴进度，不做商城消耗 |
| 已解锁反馈 | 慢眨眼、趴下、伸懒腰、小爪印 |

## 11. 页面 08：隐私与智能解释设置

### 目标

让评审看到大模型和隐私边界。

### 设置项

| 设置 | 默认 | 说明 |
|---|---|---|
| 本地行为识别 | 开 | 只统计本 App 内点击、滑动、停留 |
| 智能解释 | 关或可开 | 开启后只上传聚合特征 |
| 清除本地记录 | 可操作 | 删除会话摘要 |
| 隐私说明 | 可查看 | 不录屏、不读输入内容、不跨 App 监控 |

## 12. 调试/比赛演示层

这是演示用，不作为普通用户默认 UI。

| 组件 | 用途 |
|---|---|
| `focus_score` | 展示行为识别结果 |
| `arousal_score` | 展示活跃程度 |
| `distraction_score` | 展示分心程度 |
| 当前状态 | Normal/Transition/Focus/Reward |
| LLM 状态 | Local Template / LLM Success / Timeout Fallback |

打开方式：隐藏手势或开发版按钮，最终视频可短暂展示技术可信度。

## 13. 关键 UI 验收标准

| 编号 | 验收 | 通过标准 |
|---|---|---|
| UI-01 | 首页主题明确 | 第一屏看到猫、小镇、开始专注 |
| UI-02 | 不跑题 | 无活动、商城、抽奖、比赛重置页面 |
| UI-03 | 专注态低刺激 | 专注中不出现多余按钮和强弹窗 |
| UI-04 | 隐私可解释 | 设置页明确不录屏、不读文本、不跨 App |
| UI-05 | 视频可展示 | 首页、专注、奖励、记录均可截图进入视频/PPT |
