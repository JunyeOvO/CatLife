## 3. P0 剩余顺序

1. 在正式 Unity 工程合入 `mvp-unity-assets` 和猫 FBX。
2. 将猫咪小镇当前 no-merge 导出导入主场景，并记录性能。
3. 构建 Android Development APK。
4. 真机安装、启动、状态链录屏和 logcat。
5. 从真实画面生成 PPT 截图、视频镜头和海报主视觉。
6. 将大模型代码包模板替换为真实 provider 解析或端侧 SDK 调用说明。
7. 补 5 份用户验证摘要。
8. 运行 `tools/final-submission/check-final-submission.ps1` 生成自检表。

## 4. 自动检查工具

新增脚本：

```powershell
powershell -ExecutionPolicy Bypass -File tools/final-submission/check-final-submission.ps1
```

脚本会：

- 检查最终目录中 PPT、视频、海报、APK、代码包是否存在；
- 计算已存在提交物的 SHA256；
- 扫描常见密钥形态；
- 检查 Android 日志/安装证据是否存在；
- 输出 `06-deliverables/final-submission/CatLife_submission_check_20260705.md`。

当前预期：因为 APK、视频、PPT、海报还未生成，脚本应返回缺失项，而不是通过。
