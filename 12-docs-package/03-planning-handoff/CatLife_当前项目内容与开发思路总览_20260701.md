# CatLife 当前项目内容与开发思路总览

日期：2026-07-01
用途：作为当前仓库的高层入口文档，说明 CatLife 现在有什么、核心思路是什么、后续开发从哪里继续。旧的 2026-06-29 交接总览和本地文件盘点只保留在 Git 历史中，不再作为阅读入口。

## 1. 一句话定位

CatLife 是一款低多边形猫咪陪伴式专注 App。它不把用户直接锁住，而是通过非敏感行为特征识别用户是否正在从分心走向专注，再让猫咪、小镇、轻锁定界面和鼓励气泡共同完成“温和进入专注”的引导。

核心表达：

- 不是传统番茄钟：先观察状态，再引导专注。
- 不是强制锁机：保留用户退出权，降低抵触。
- 不是普通宠物 App：猫咪行为由专注状态驱动，承担心理反馈功能。
- 不是活动/抽奖/社交产品：比赛版本只做专注闭环和可信演示。

## 2. 当前产品闭环

```text
用户打开 App
  -> 看到猫咪小镇和猫
  -> 点击开始专注
  -> 行为识别进入 Transition / Focus
  -> 猫咪降低活跃度、气泡提示或鼓励
  -> 轻锁定界面保持低打扰
  -> 完成专注后进入 Reward
  -> 生成记录、成长反馈和可解释总结
```

MVP 状态机仍保持四态：

| 状态 | 用户状态 | 猫咪表现 | UI 表现 | 大模型/规则作用 |
|---|---|---|---|---|
| Normal | 普通浏览或刚进入 App | 走动、抬头、轻互动 | 主界面按钮完整 | 本地规则记录基础行为 |
| Transition | 操作频率下降，可能准备专注 | 看向用户、靠近、轻提示 | 弱提醒和气泡 | 判断是否继续引导 |
| Focus | 低频操作持续 | 安静、陪伴、低动作幅度 | 轻锁定、计时、退出手势 | 可用 LLM 生成解释/鼓励 |
| Reward | 专注完成 | 摇尾、庆祝、奖励反馈 | 记录、成长、总结 | 生成专注反馈文案 |

## 4. 当前开发思路

### 4.1 先做真实 App，不再继续堆概念稿

后续优先级是 Unity/Android 可运行闭环，而不是继续扩写视觉方案。目标是先拿到一屏可录制的真实画面：

```text
猫咪小镇场景 + 猫模型 + 固定高度摄像机 + 基础 UI Overlay + 开始专注按钮
```

只要这个画面能在真机上稳定运行，PPT、视频、海报和答辩材料才有可信截图。

### 4.2 最新本地资产为基线，旧 Unity 只作参考

协作者 Unity 内容和 `work/unity-android-build-batch-20260629/` 的价值是：

- 参考已有脚本命名、状态机拆分和场景搭建方式。
- 查阅 Unity 包、ProjectSettings、UI 图片和 TextMesh Pro 资源。
- 对照旧实现，避免重新踩坑。

它们不应被全量复制成最终 App，因为旧场景不是从最新猫咪小镇和当前猫动画资产重建的，可能带入旧引用、旧相机、旧 UI 视觉和非必要依赖。

### 4.3 小镇在 Blender 收口，天空和运行时表现放到 Unity

Blender 负责：

- 小镇几何、材质基色、低多边形风格。
- 空岛底座、草地、树木、围栏、建筑、道具层级。
- 导出前清理、预览图和源文件回退。

Unity 负责：

- Skybox / Directional Light / 环境光。
- 移动端阴影、材质压缩、批处理和相机。
- Prefab 分组、碰撞、交互锚点和运行时优化。

这样做比在 Blender 中烘死一个半球天空盒更稳，也更容易适配移动端。

### 4.4 UI 是叠加层，不应遮挡场景主体

无 UI 的基础画面应只有：

```text
猫咪 + 猫咪小镇背景
```

UI 作为 Canvas Overlay 叠加：

- 顶部：品牌、今日专注时间、当前状态。
- 右侧：猫咪、记录、专注森林、设置等入口。
- 中部：猫咪气泡，按状态短句提醒。
- 底部：开始专注 / 退出专注 / 状态切换。
- 摄像机控制：固定高度旋转和前后移动，旋转为原地 360 度。
